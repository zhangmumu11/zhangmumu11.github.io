docker学习之路

2020-04-14 Comments

###容器保存为镜像

docker 备份与迁移

commit 将容器保存为镜像

docker commit (mynginx)容器名称 （mynginx_i）镜像名称

###镜像备份

save -o 将将进行镜像保存为tar文件

dockers save -o mynginx.tar mynginx_i

###镜像的恢复与迁移

首先删掉mynginx镜像 rm

然后执行命令进行回复

docker load -i mynginx.tar(tar文件名称)

load 加载镜像

-i 输入文件

docker file

基于一个基础进行镜像来构建一个新的镜像

基础镜像：系统级别的镜像 centos ubuntu

通过一个dockerfile文件脚本文件来构建一个完全一致的环境

1.开发人员：开发环境提供完全一直的开发环境

2.测试人员：直接拿开发人员的镜像或者通过dockerfile文件构建一个新的镜像直接进行测试

3.运维人员：无缝移植

常用命令

FROM image_name:tag 定义基础镜像启动构建流程 必须

MAINTANER user_name 声明镜像的构建者

ENV key value 设置环境变量 （可以写多条)

RUN command 是Dockerfile核心部分（可以写多条）

创建文件，编写配置文件等等

ADD source_dir/file dest_dir/file 将宿主机的文件复制到容器中，如果是一个压缩文件

，将会在复制同时自动解压

COPY source dir/file dest_dir/file 将宿主机的文件复制到容器中，只是复制，不解压

WORFDIR path_dir 设置工作目录 设置相对路径 所有的命令在当前目录中执行

创建目录之后 登录也在此目录

例：

使用centos7 构建一个jdk8的镜像

创建一个目录

mkdir -p /usr/local/dockerjdk8

将jkd8的压缩文件上传到当前文件夹

编写dockerfile

文件名称 Dockerfile 固定

基础镜像

FROM centos:7

创建者名称

MAINTAINER zhq

工作过目录

WORKDIR /usr

创建 /usr/local/java
RUN mkdir /usr/local/java

复制并解压jdk-8u171-linux-x64.tar.gz 目标目录/usr/local/java/

命令行为当前目录 如果不是写全局路径

ADD jdk-8u171-linux-x64.tar.gz /usr/local/java/

配置环境变量

ENV JAVA_HOME /usr/local/java/jdk1.8.0_171

配置环境变量

ENV JRE_HOME 美元符号JAVA_HOME/jre

配置环境变量 : 用冒号分隔 美元符号$

ENV CLASSPATH 美元符号JAVA_HOME/bin/dt.jar :

美元符号JAVA_HOME/lib/toos.jar :

美元符号JAVA_HOME/lib:CLASSPATH

ENV PATH 美元符号JAVA_HOME:/bin:$PATH

####通过dockerfile构建镜像

-t 指定构建名称

. 指定当前的文件夹 如果不是写全局路径

-f jdkdockerfile 也可以加-f 指定dockerfile文件 这样dockerfile名称不用固定

docker build -t ‘jdk1.8’ .
