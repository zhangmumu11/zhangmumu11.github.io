docker软件部署

应用部署

mysql部署

1.拉取镜像

docker pull mysql

2.创建容器

docker run -di –name=mysql -p 3306:3306 -e MYSQL_ROOT_PASSWORD=123456 mysql

-p 代表端口映射 格式为 宿主机IP：容器运行ip

-e 代表添加环境变量 MYSQL_ROOT_PASSWORD 是root用户的登录密码

3.进入sql容器

docker exec -it mysql /bin/bash

4.登录mysql

mysql -u root -p

5.远程登录mysql

tomcat部署

1.拉取镜像

docker pull tomcat

2.创建容器

docker run -di –name=mytomcat -p 8081:8080 -v /usr/local/webapps:/usr/local/tomcat/webapps tomcat

-p 代表端口映射 格式为 宿主机IP：容器运行ip

-v 代目录映射

3.进入sql容器

docker exec -it mytomcat /bin/bash

nginx部署

1.拉取镜像

docker pull nginx

2.创建容器

docker run -di –name=mynginx -p 80:80 nginx

-p 代表端口映射 格式为 宿主机IP：容器运行ip

主要修改config文件

redis部署

1.拉取镜像

docker pull redis

2.创建容器

docker run -di –name=myredis -p 6379:6379 redis

-p 代表端口映射 格式为 宿主机IP：容器运行ip

主要修改config文件

docker常见命令

docker 学习过程笔记

第一章docker 镜像命令

拉取 docker pull 镜像名称

删除 docker rmi imageID（镜像id）

查看 docker images

第二章docker容器命令

docker ps 查看运行中的

docker ps -a 查看所有的容器

1.创建与启动容器

创建容器 docker run

-i 运行的容器

-t 创建容器之后进去其命令行，加入-it之后，及分配一个伪客户端

-v 表示目录映射关系（java：java）前者是宿主机目录，后者是映射到宿主机的目录即为容器目录

可以使用多个-v 做多个文件或目录的映射。最好做目录映射，在宿主机上修改，然后共享到容器上。

-d 创建一个守护式进程的容器在后台运行（这样创建容器不会自动登录容器，如果加上-it 参数，则会自动进入容器）

-p 表示端口映射，前者是宿主机端口，后者是容器内映射的端口，可以使用多个-p 做多个端口映射。

2.交互式创建容器

docker -it –name=mycentos centos:7 /bin/bash

创建完之后即进入容器内部

3.守护式创建容器

docker run -di –name=mycentos2 centos

创建完之后出现很长一段字符串则为创建成功

a830d8160184e2ea71de07dc81fa3bb12963ded775cadbf0b6874b6a5a5eac82

创建完之后则不会进入容器

4.登录守护式容器

docker exec -it 容器名称 /bin/bash

进入容器中的/bin/bash 目录

5.退出容器

容器中 exit

交互式创建的容器在退出之后即关闭容器

守护式不会

6.停止与启动容器

6.1停止容器

docker stop 容器名称（或者容器id）

6.2启动容器

docker start 容器名称（或者容器id）

7.文件拷贝

将文件拷贝到容器mycentos中/usr/local目录中

docker cp 文件当前的路径 mycontos:/usr/local

将docker容器中的文件任意拷贝到宿主机

docker cp mycontos:/usr/local 文件路径和名称

8.目录挂载

将容器中的目录映射到宿主机上的目录，这样就可以通过修改宿主机上的文件来修改容器中的文件

加上 -v 参数

–docker run -di –name=mycentos2 -v /usr/local/myhtml:/usr/local/myhtml centos

9.查看容器的ip地址

docker inspect 容器名称（容器id）

查看容器的所有新信息

格式化输出想要的数据

–docker inspect –format=.NetworkSettings.IPAddress mycentos2

format中的参数用大括号括起来

10.删除容器

–docker rm 容器名称（容器id）

删除之前必须停止容器 docker stop 容器名称（容器id）
